HappyBags Pro Manager Test Plan Document

HappyBags Pro Manager - Test Plan

Comprehensive testing checklist for all application features

Test Execution Notes:
• Test with both Manager and Sales Rep roles
• Always test: Happy path + Edge cases + Error handling
• Ensure test data is prepared before testing

1. AUTHENTICATION & AUTHORIZATION

1.1 Login/Logout

  • Login with valid manager credentials
  • Login with valid sales rep credentials
  • Login with invalid credentials (wrong password)
  • Login with non-existent user
  • Logout redirects to login page
  • Access protected route without login redirects to login

1.2 Role-Based Access Control (RBAC)

  • Manager can access /reports
  • Manager can access /audit
  • Manager can access /users
  • Sales rep CANNOT access /reports (redirected or error)
  • Sales rep CANNOT access /audit (redirected or error)
  • Sales rep CANNOT access /users (redirected or error)
  • Sales rep CAN access /orders and /dashboard

2. DASHBOARD

2.1 Statistics Cards

  • Total Orders count is accurate
  • Total Clients count is accurate
  • Total Products count is accurate
  • Total Revenue displays correct sum (GHC)
  • Pending Orders shows correct count
  • Completed Orders shows correct count

2.2 Charts

  • Monthly Sales Trend chart renders
  • Orders by Status pie chart renders
  • Revenue by Category bar chart renders
  • Charts update when period filter changes (Today/Week/Month/Quarter)

2.3 Quick Actions & Recent Orders

  • "New Order" button navigates to order creation
  • "Add Client" button opens client modal
  • "Add Product" button opens product modal
  • "View Reports" navigates to reports page
  • Recent Orders list shows last 5 orders
  • Clicking recent order navigates to order detail

2.4 Keyboard Shortcuts

  • / focuses search bar
  • N opens new order modal
  • C opens add client modal
  • P opens add product modal
  • D navigates to dashboard
  • ? shows keyboard shortcuts help

3. ORDERS MANAGEMENT

3.1 Order List

  • All orders display in table
  • Search by order ID works
  • Search by client name works
  • Filter by status works (Active/All/Completed)
  • Expandable row shows order items
  • Status badges show correct colors

3.2 Order Status Workflow

  • New order starts at "Estimate Offered" status
  • "Advance Status" button moves to next stage
  • Order reaches "Order Completed" final state
  • Status history shows all transitions

3.3 Create Order

  • Select existing client from dropdown
  • Create new client during order (quick add)
  • Add multiple products to order
  • Quantity adjustment updates total
  • Discount (percentage) applies correctly
  • Discount (fixed amount) applies correctly
  • VAT toggle adds/removes 20%
  • Delivery date picker works
  • Delivery time picker works
  • Subtotal calculates correctly
  • Total calculates correctly
  • Order ID generates in format HPBG-YYYY-XXXX

3.4 Edit Order

  • Edit order items (add/remove/update quantity)
  • Edit client
  • Edit delivery date/time
  • Edit discount
  • Toggle VAT
  • Changes save correctly

3.5 Order Actions

  • View payment history
  • Add payment modal opens
  • Generate Estimate PDF
  • Generate Proforma PDF
  • Generate Invoice PDF
  • Generate Receipt PDF
  • Preview documents
  • Archive order (soft delete)
  • View status history
  • Revert status (manager only)

3.6 Archived Orders

  • View archived orders list
  • Restore archived order
  • Permanently delete order (manager only)

4. PAYMENTS

4.1 Record Payment

  • Add payment to order
  • Enter amount (full or partial)
  • Select payment type (Hubtel/Other)
  • Payment saves to order
  • Order status auto-advances when fully paid

4.2 Payment Tracking

  • Total Paid updates after payment
  • Outstanding amount calculates correctly
  • Payment history shows all payments

4.3 Delete Payment (Manager Only)

  • Delete payment removes it from order
  • Order totals recalculate correctly

5. PRODUCTS

5.1 Product List

  • All products display in grid
  • Search by product name works
  • Search by category works
  • Search by dimension works

5.2 CRUD Operations

  • Add new product (all fields)
  • Edit existing product

5.3 Product Categories

7.2 Report Generation

8.1 Viewing Logs

  • All activities display in list
  • Filter by entity type works
  • Filter by action works

8.2 Log Details

  • User who performed action is shown

  • IP Address is captured
  • Old values shown for updates
  • New values shown for updates

9. USER MANAGEMENT (Manager Only)

9.1 User List

  • All users display
  • User roles shown correctly

9.2 CRUD Operations

  • Create new user
  • Assign role (Manager/Sales Rep/Client)
  • Edit user details
  • Delete user (cannot delete last manager)
  • Link client to user account

10. PROFILE

  • View profile information
  • Update name
  • Update email
  • Update password
  • Delete account

11. EDGE CASES & ERROR HANDLING

11.1 Orders

  • Create order with zero products (should fail validation)
  • Create order without client (should fail)
  • Advance status when already complete (button disabled)
  • Negative quantity (should be prevented)
  • Negative discount (should be prevented)

11.2 Products

  • Negative price (should be prevented)
  • Delete product used in orders (manager only - allow but check impact)

11.3 Clients

  • Duplicate client name (allow but warn)
  • Delete client with orders (should fail with message)

11.4 Payments

  • Payment amount exceeds order total (should warn)
  • Negative payment amount (should fail)

11.5 Reports

  • Date range with no data (show "No data available")
  • Invalid date range (to < from - should fail validation)

12. RESPONSIVE DESIGN

  • Desktop layout renders correctly
  • Tablet layout renders correctly
  • Mobile layout renders correctly
  • Sidebar collapses on mobile
  • Bottom navigation appears on mobile
  • All buttons are touch-friendly on mobile
